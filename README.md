# Vestigra

> 当意外发生，你的 Mac 仍记得经过。  
> When something goes wrong, your Mac remembers.

Vestigra 是一个本地优先的 macOS 安全记忆工具，帮助你看清 Mac 上发生过什么：哪个文件被下载，谁执行了它，它改了什么，有没有写入后台启动项，有没有第一次运行后联网，以及这些判断背后有哪些证据。

它不是杀毒软件，也不是企业安全平台。  
Vestigra 更像一个给个人开发者和高安全意识用户准备的本地安全黑匣子。

官网：<https://vestigra.xiaodiao.tech>

## Vestigra 能帮你做什么？

### 看懂一次安装到底发生了什么

当你安装一个工具、打开一个 DMG、运行一个脚本，或执行一次 `npm install` 时，Vestigra 可以帮你回答：

- 这个文件是从哪里来的？
- 谁第一次执行了它？
- 它后面又启动了哪些进程？
- 它有没有写入后台启动项？
- 它第一次运行后有没有联网？
- 这次行为为什么值得注意？

不用从零散日志里捞线索，Vestigra 会把相关行为整理成更容易理解的故事。

---

### 追踪开发者供应链风险

开发者每天都在安装依赖、运行脚本、下载工具、更新插件。大多数时候一切正常，但当你想知道“刚才那个包到底做了什么”时，Vestigra 可以提供更清楚的答案。

典型场景：

- npm / pnpm / yarn 安装脚本执行后发生了什么
- 新生成的可执行文件第一次运行后连接了哪里
- 安装器是否写入了 LaunchAgent 或 LaunchDaemon
- 新 helper、插件或脚本是否在后台持续运行
- 某个陌生网络连接是由哪个进程触发的

---

### 把异常变成可读的行为故事

Vestigra 不只展示事件列表。它会尝试把行为串起来：

```text
浏览器下载了安装包
用户打开了安装器
安装器运行了脚本
脚本创建了后台 helper
helper 写入了启动项
helper 第一次运行后连接了外部地址
```

你可以从故事里继续展开证据，查看每一步的来源和不确定性。

---

### 告诉你哪里看见了，哪里没看见

安全工具最危险的幻觉是“没看到，所以没发生”。

Vestigra 会尽量明确区分：

- 已观察到的行为
- 根据关系推断出的线索
- 证据不足的部分
- 采集或覆盖缺口

它不会假装全知。它的目标是让你更清楚地判断，而不是给你一个过度自信的结论。

---

### 用可审查规则提醒你关心的行为

你可以把自己关心的问题表达成规则，例如：

```text
如果安装脚本生成了新的可执行文件，
并且它第一次运行后连接了陌生外部地址，
就提醒我。
```

Vestigra 的目标不是让 AI 自动替你决定一切，而是让规则可以被解释、审查和调整。

---

### 默认本地，默认只关注安全元数据

Vestigra 的默认原则：

- 默认本地
- 只关注安全相关元数据
- 不默认上传
- 不默认采集内容
- 用户同意才导出
- 高风险动作需要用户确认

Vestigra 关注的是“发生了什么安全相关行为”，不是读取你的文档、聊天、邮件或网页内容。

## 当前适合谁？

Vestigra 当前优先适合：

- 软件开发者
- 独立开发者
- 安全意识强的 Mac 用户
- macOS 安全研究者
- 需要本地案件导出的轻量调查场景

尤其适合你经常做这些事：

- 安装 npm / pnpm / yarn 依赖
- 下载 DMG 或安装器
- 运行第三方脚本
- 安装开发工具、插件或 helper
- 想知道 Mac 上某次异常变化从哪里开始

---

## Vestigra 不是什么？

Vestigra 不是：

- 杀毒软件
- 防火墙
- MDM 或远程管理工具
- 员工监控软件
- 自动清除恶意软件的工具
- 自动阻止所有攻击的工具
- 绝对防篡改系统

Vestigra 的目标是记录、解释、提醒和辅助调查，而不是替用户做所有高风险决定。
